# 📋 Frontend

ТЗ прямо требует «интуитивный интерфейс: исследователь без подготовки формулирует запрос на естественном языке и получает понятный ответ» + «визуализация графа».

## Что строим (3 экрана в одном)
1. **Поле вопроса** — юзер пишет запрос обычным языком, жмёт «Спросить».
2. **Ответ** — связный текст + **список источников** (кликабельные) + **уровень уверенности**. Если данных нет — честный блок «Пробел в данных».
3. **Визуализация подграфа** — узлы (материалы, процессы, свойства…) и связи между ними по теме запроса. С **подсветкой**: 🔴 противоречия, 🟡 пробелы. Это наша фишка — сделай её заметной.

## Стек (на твоё усмотрение)
- React  + `fetch` к нашему API.
- Граф: **cytoscape.js** или **vis-network** — обе умеют рисовать узлы/рёбра и раскрашивать. Не изобретай визуализацию с нуля.


## Контракт API (предварительный, точнее согласуем сегодня)
Работай пока на этих моках, потом просто поменяешь URL на наш бэкенд:
```
POST /ask
  тело:   { "question": "строка запроса" }
  ответ:  {
    "answer": "связный текст ответа",
    "sources": [ { "doc_id": "...", "title": "...", "year": 2024, "snippet": "..." } ],
    "confidence": "high | medium | low",
    "graph": {
      "nodes": [ { "id": "n1", "label": "Никель", "type": "Material" }, ... ],
      "edges": [ { "from": "n1", "to": "n2", "label": "uses_material", "flag": "normal|contradiction|gap" }, ... ]
    },
    "gaps": [ "нет экспериментов для: холодный климат + кучное выщелачивание + никель" ],
    "contradictions": [ { "about": "...", "sources": ["docA","docB"] } ]
  }
```
Типы узлов (для раскраски): `Material, Process, Equipment, Property, Experiment, Publication, Expert, Facility`.

## Что делать УЖЕ СЕЙЧАС (не ждёт данных)
1. Каркас страницы: поле ввода + область ответа + область графа.
2. Замокай ответ (`mock.json` по формату выше) и отрисуй по нему: текст, источники, подсветку узлов по `type`, а рёбра — по `flag` (красный = противоречие, жёлтый = пробел).
3. Проверь, что граф читаемо выглядит на 15–30 узлах.

Когда бэкенд будет готов — просто заменишь мок на реальный `POST /ask`.
