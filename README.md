# Научный клубок — граф знаний R&D для горно-металлургической отрасли

Система, которая собирает разрозненные научные и технические документы (статьи, отчёты,
обзоры, патенты, справочники) в единый **граф знаний** и позволяет задавать вопросы на
естественном языке. В ответ система даёт связный текст **со ссылками на источники**,
показывает связанный подграф сущностей и — главное — подсвечивает **противоречия и пробелы
в данных**.

Трек «Научный клубок», Норникель AI Science Hack 2026.

## Задача

Знания в R&D рассеяны по сотням документов: что уже пробовали, какие данные есть, какие
выводы подтверждены, где противоречия, кто носитель экспертизы. Из-за этого команды дублируют
исследования, медленно принимают решения и получают противоречивые выводы.

Нужна система, которая связывает разнородные сущности (публикации, эксперименты, материалы,
процессы, оборудование, свойства, эксперты) и отвечает на запросы вида:

- «Какие методы обессоливания воды подходят, если исходная вода содержит сульфаты, хлориды,
  Ca, Mg, Na по 200–300 мг/л, а требуемый сухой остаток ≤1000 мг/дм³?»
- «Какие технические решения циркуляции католита при электроэкстракции никеля описаны в
  мировой практике и какая скорость потока считается оптимальной?»
- «Покажите эксперименты и публикации по распределению Au, Ag и МПГ между штейном и шлаком
  за последние 5 лет».

## Что делает система

- **Импорт и нормализация** разнородных документов (PDF, DOCX) в единый формат.
- **Извлечение знаний**: из текста вытягиваются сущности (материалы, процессы, оборудование,
  свойства…), связи между ними и **числовые ограничения с единицами** (например, «сульфаты
  ≤300 мг/л», «температура 80 °C»).
- **Граф знаний** по онтологии домена (типы: Material, Process, Equipment, Property,
  Experiment, Publication, Expert, Facility; связи: uses_material, produces_output,
  described_in, validated_by, contradicts и др.).
- **Поиск и ответы**: гибридный поиск (по смыслу + по ключевым словам) + генерация связного
  ответа по найденным фрагментам, с указанием источников и уровня уверенности.
- **Фильтры**: по материалу, процессу, географии (отечественная / зарубежная практика),
  году, числовым диапазонам.
- **Аналитика (ключевая возможность)**:
  - **Пробелы в знаниях** — какие комбинации «материал × процесс × условие» не исследованы
    (например: «нет данных для: холодный климат + кучное выщелачивание + никелевая руда»).
  - **Противоречия** — где разные источники дают конфликтующие выводы или числа по одной теме.

## Как это работает

```
Документы → Парсинг/нормализация → Извлечение сущностей, связей, чисел (LLM)
                                              │
                     ┌────────────────────────┴───────────────────┐
                     ▼                                             ▼
           Граф знаний (networkx)                        Векторный индекс (FAISS)
                     └────────────────────┬────────────────────────┘
                                          ▼
                       Поиск (граф + вектор) → ответ с источниками (RAG)
                                          ▼
                       Анализ: пробелы и противоречия → подсветка
                                          ▼
                            API (FastAPI) → веб-интерфейс
```

## Технологии

- **Язык**: Python 3.13
- **Граф**: networkx (за интерфейсом `GraphStore` — при необходимости заменяется на Neo4j)
- **LLM**: Yandex AI Studio (yandexgpt) / GigaChat — за интерфейсом `LLMClient`
- **Эмбеддинги**: bge-m3 (локально) / Yandex Embeddings
- **Векторный поиск**: FAISS
- **API**: FastAPI
- **Фронтенд**: веб-интерфейс с визуализацией графа

## Структура репозитория

```
ingest/     — парсинг документов в единый формат documents.jsonl
core/       — llm, извлечение сущностей, граф, индекс, поиск+RAG, анализ (пробелы/противоречия)
api/        — HTTP-API (FastAPI)
frontend/   — веб-интерфейс
scripts/    — сборка: данные → граф + индекс
data/       — данные (не в git): raw/ (корпус), processed/ (артефакты), sample/ (мини-набор)
```

## Установка и запуск

Требуется Python 3.13.

```bash
# 1. Окружение и зависимости
python -m venv .venv
.venv\Scripts\activate            # Windows;  source .venv/bin/activate — Linux/Mac
pip install -r requirements.txt

# 2. Ключи (скопировать .env.example → .env и заполнить)
#    YC_API_KEY, YC_FOLDER — доступ к Yandex AI Studio
#    (опционально GIGACHAT_CREDENTIALS — запасная модель)

# 3. Данные: положить корпус в data/raw/  (или использовать data/sample/ для быстрой проверки)

# 4. Сборка графа и индекса
python scripts/build.py --input data/raw --output data/processed

# 5. Запуск API
uvicorn api.main:app --reload

# 6. Фронтенд
cd frontend && cp .env.example .env && npm install && npm run dev
```

Переменные фронтенда — в `frontend/.env` (шаблон: `frontend/.env.example`):
- `VITE_API_URL` — базовый URL API (`/api` в dev, полный URL бэкенда на деплое)
- `VITE_API_PROXY_TARGET` — адрес бэкенда для dev-прокси Vite (по умолчанию `http://localhost:8000`)

Готовые артефакты (обработанный `documents.jsonl`, граф, индекс) для полного корпуса
выложены на облачный диск — ссылка в материалах сдачи, чтобы не пересобирать 5 ГБ вручную.
