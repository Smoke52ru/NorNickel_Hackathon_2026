# NorNickel_Hackathon_2026
НОРНИКЕЛЬ AI SCIENCE HACK - Научный клубок

## Брифинг
Ваша задача: создать knowledge graph или поисково-аналитическую систему, которая связывает статьи, эксперименты, материалы, свойства, режимы, установки, исследовательские команды и выводы. Вам не нужно вписываться в жесткие рамки — вы сами решаете, какую архитектуру и интерфейс предложить: графовую БД, семантический поиск, гибридное решение. Главное — чтобы система отвечала на вопросы вида: «что уже делали по сплавам X при режиме Y и какой был эффект на свойство Z», показывала связанные сущности, историю решений и пробелы в данных. Для работы вы получите доступ к корпусу внутренних документов, каталогу экспериментов, справочникам материалов и оборудования, перечню сотрудников/лабораторий и тегам тематик.

## Структура проекта

Проект организован как fullstack приложение с разделением на frontend и backend:

```
NorNickel_Hackathon_2026/
├── frontend/          # React + Vite приложение (наша часть)
│   ├── src/
│   │   ├── components/   # Переиспользуемые UI компоненты
│   │   ├── pages/        # Страницы приложения
│   │   ├── hooks/        # Кастомные React hooks
│   │   ├── services/     # API сервисы для работы с бэкендом
│   │   ├── types/        # TypeScript типы и интерфейсы
│   │   ├── utils/        # Утилитарные функции
│   │   ├── App.tsx       # Корневой компонент
│   │   └── main.tsx      # Точка входа приложения
│   ├── public/         # Статические ресурсы
│   ├── index.html      # HTML шаблон
│   ├── package.json    # NPM зависимости
│   ├── tsconfig.json   # Конфигурация TypeScript
│   └── vite.config.ts  # Конфигурация Vite
│
├── backend/           # Python + FastAPI (не наша часть работы)
│   # Здесь будет размещаться backend приложение
│   # Разрабатывается отдельной командой
│
└── README.md          # Документация проекта
```

### Frontend структура

- **`components/`** — Переиспользуемые React компоненты (карточки, графы, формы поиска)
- **`pages/`** — Компоненты страниц, соответствующие маршрутам приложения
- **`hooks/`** — Кастомные React hooks для бизнес-логики
- **`services/`** — Модули для работы с API (Axios инстанс, функции запросов)
- **`types/`** — TypeScript типы для сущностей knowledge graph (Entity, Relationship и др.)
- **`utils/`** — Вспомогательные функции (форматирование, валидация)

### Backend структура

Папка `backend/` зарезервирована для команды backend-разработки. 
**Важно:** Мы не модифицируем эту папку — это зона ответственности другой команды.

## Технологии

### Frontend Stack

| Технология | Назначение |
|------------|------------|
| **React 18** | UI библиотека для построения интерфейса |
| **TypeScript** | Типизация для безопасности кода |
| **Vite** | Сборщик и dev-сервер (быстрая HMR) |
| **React Router** | Маршрутизация в SPA |
| **Axios** | HTTP клиент для API запросов |

### Backend Stack (справочно)

| Технология | Назначение |
|------------|------------|
| **Python** | Язык программирования backend |
| **FastAPI** | Веб-фреймворк для API |

### Взаимодействие Frontend ↔ Backend

- Frontend запускается на `localhost:3000` (Vite dev server)
- Backend работает на `localhost:8000` (FastAPI)
- Vite проксирует запросы с `/api/*` на backend сервер
- Формат обмена данными: JSON REST API
