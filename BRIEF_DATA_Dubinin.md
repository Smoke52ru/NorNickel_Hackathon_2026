# 📋 Data Engineer

 Ты отвечаешь за **самую фундаментную часть** — превращение хаоса из документов в чистые, структурированные данные, на которых работает вся система. Без тебя ни граф, ни поиск не поедут.

## Твоя цель одним предложением
Взять разнородные документы (PDF, docx, архивы) и выдать **единый машиночитаемый формат** — один JSON-объект на документ (или на его смысловой кусок), с текстом и метаданными, сохраняя **откуда** это взято (провенанс критичен — на нём строится наша фишка «источник + достоверность»).

## Данные, с которыми работаешь
Папка `Источники информации`. Это наши основные данные, работаем прямо по ним:
- Журналы «Горная промышленность»/«Горный журнал» (2024–2026), материалы конференций, доклады, обзоры, статьи. ~1448 файлов, 4.9 ГБ. В основном PDF и docx, есть zip/rar.
- **🎯 Папка `Обзоры` (docx) — ПРИОРИТЕТ №1.** Это готовые литобзоры прямо по темам вопросов из ТЗ: «Методы очистки шахтных вод», «Обзор технических решений электролитического производства никеля и меди», «Зарубежный и отечественный опыт флотации шлаков» и т.д. На них проще всего извлекаются сущности, связи И противоречия. Начни отсюда — так демо сразу отвечает на вопросы-примеры.
- **46 xls/xlsx — это рыночная статистика** (Brook Hunt/CRU/ICSG: медь, МПГ, кобальт — производство/цены). Второстепенно, для «экономических показателей». Пока пропусти.

## Целевой формат вывода (это формат работы, точнее согласуем сегодня позже, используй его)
Файл `documents.jsonl` — по одному JSON на строку:
```json
{
  "doc_id": "gp_2024_03_art12",
  "source_path": "Журналы/Горная промышленность/2024/ГП № 3-2024.pdf",
  "source_type": "journal",         // journal | conference | report | review | article | reference
  "title": "…",
  "authors": ["…"],                  // если удаётся вытащить
  "year": 2024,                       // из имени файла/пути/текста
  "lang": "ru",                       // ru | en (определять автоматически)
  "text": "полный чистый текст…",
  "chunks": ["кусок 1", "кусок 2"],  // нарезка на смысловые куски (см. ниже)
  "meta": {"journal": "Горная промышленность", "issue": "3-2024"}
}
```

## Что изучить (концепции и инструменты)
- **Извлечение текста из PDF:** `PyMuPDF` (он же `fitz`) — быстрый и качественный, ИЛИ `pdfplumber` (лучше для таблиц). В нашем корпусе PDF в основном с **текстовым слоем** (не сканы) — значит тяжёлый OCR НЕ нужен. Но проверь: если попадётся скан (пустой текст) — такой файл пока пропускай и логируй, не застревай.
- **docx:** `python-docx`. Для старых `.doc` — можно через `textract` или конвертацию, но если их мало — пропусти, не трать время.
- **Архивы (zip/rar):** распаковывать (`zipfile`, для rar — `patool`/`rarfile`) и рекурсивно обрабатывать содержимое.
- **xls/xlsx:** `pandas.read_excel` — это, скорее всего, каталоги/справочники (материалы, эксперименты). Их структуру сохраняй отдельно — они пойдут прямо в граф почти без NLP.
- **Определение языка:** `langdetect` или `fasttext-langid` — нужно для рус/англ.
- **Нарезка на чанки (chunking):** режь текст на куски ~500–800 слов с перекрытием ~100 слов, по возможности по границам абзацев/разделов. Зачем: длинный документ целиком не влезет в модель, а поиск идёт по кускам.
- **Дедупликация:** одинаковые/почти одинаковые документы отсекай (по хешу текста или заголовку) — в корпусе бывают повторы.

## Идеи и приоритеты (важно!)
1. **НЕ обрабатывай все 4.9 ГБ.** Это ловушка. Возьми **релевантное подмножество** под нашу тему (металлургия/гидрометаллургия/обогащение/очистка вод/электроэкстракция) — 50–150 документов для начала более чем достаточно, чтобы система заработала и демо было живым. Полный прогон — потом, если останется время.
2. Всегда сохраняй `source_path` и метаданные.
3. **Роботоспособность важнее полноты.** Один битый файл не должен ронять весь прогон: оборачивай обработку каждого файла в try/except, логируй ошибки в отдельный файл, иди дальше. Лучше 120 хорошо разобранных документов, чем падение на 5-м.
4. **Метаданные из пути/имени.** Год, журнал, номер выпуска часто прямо в пути (`Журналы/Горный журнал/2024/№ 03_24.pdf`) — вытаскивай регуляркой, это дешёвые и ценные метаданные (для фильтра по времени из ТЗ).
5. **Таблицы (xls) — отдельным треком.** Справочники и каталоги экспериментов почти структурированы — их можно почти напрямую класть в граф. Это самый «дешёвый» и надёжный источник качественных сущностей. Начни с них — быстрый результат.
6. **Выдай первую партию КАК МОЖНО РАНЬШЕ.** Даже 20 документов в `documents.jsonl` разблокируют всю остальную команду. Не полируй месяц — итерируй.

## Чего НЕ делать
- Не пиши OCR-пайплайн (текстовый слой есть).
- Не строй сам граф и не извлекай сущности LLM-ом — это моя часть. Твой выход — чистый текст + метаданные.
- Не гонись за идеальным парсингом таблиц внутри PDF — если сложно, вытащи хотя бы текст.
